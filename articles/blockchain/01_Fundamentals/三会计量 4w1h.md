# 三会计量（cash / borrows / reserves）

`cash`、`borrows`、`reserves` 是借贷协议中最核心的三项会计量，用于判断流动性、风险与收入缓冲能力。

## What（是什么）

借贷协议中衡量资金状态的三大核心指标。

| 指标 | 含义 |
| --- | --- |
| **cash** | 协议当前可用的资产余额（在合约里的可立即借出的资金） |
| **borrows** | 所有借款人尚未偿还的总借款额（含利息累积） |
| **reserves** | 协议积累的储备金，用于风险缓冲和坏账处理 |

## Why（为什么重要）

- **健康度评估**：判断协议是否有足够现金应对提款和清算
- **风险分析**：borrows 占比过高说明流动性风险上升
- **收入来源**：reserves 是协议收入和安全缓冲的体现
- **验证 RPC 调用**：在 fork 调试中读取这三项可以验证 ABI、RPC、环境是否正确

## When（什么时候用）

- **日常监控**：链上分析/数据看板持续跟踪协议运行情况
- **清算前后对比**：判断一次清算对协议资金结构的影响
- **策略回测**：在历史区块 fork 中计算收益和风险
- **利率模型测试**：根据 cash/borrows 计算利用率（Utilization）

## Where（在哪获取）

### 链上调用

```solidity
getCash()
totalBorrows()
totalReserves()
```

### 接口来源

- 借贷市场合约（Compound cToken、Comet、Aave Pool 等）
- 区块浏览器（Etherscan Read Contract）
- Dune / The Graph 数据接口

## How（怎么用）

核心规则：优先监控利用率与储备缓冲，先看 `borrows / (cash + borrows)`，再看 `reserves` 是否足以覆盖异常波动。

- **计算利用率**：`Utilization = borrows / (cash + borrows)`
- **估算可清算规模**：根据 cash 判断协议短期内可供借出/清算的额度
- **监控健康度**：结合 reserves 看协议应对坏账的能力
- **调试脚本**：在 fork 模式下调用三会计量确认链上状态是否和预期一致
