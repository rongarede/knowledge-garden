# 区块链求职计划

本清单用于管理区块链求职的 4 周执行节奏，从定位、投递到面试复盘，目标是稳定提升回复率与通过率。

建议每天固定时间更新投递记录，并在每周末做一次策略复盘。

## 第0周：定位 & 材料准备

- [ ] 梳理目标岗位（研发 / 研究 / 产品 / 运营）
- [ ] 输出目标岗位技能清单（Solidity、Rust、共识协议、ZK、区块链生态…）
- [ ] 完善简历（针对不同岗位准备 2-3 个版本）
- [ ] 优化 LinkedIn & BOSS直聘 资料
- [ ] GitHub 上传代表性项目（LightDAG 实验、区块链仿真代码）
- [ ] 准备一份自我介绍（中英文版，2分钟内）

## 第1周：快速投递

- [ ] 每天投递 3–5 个岗位（总计 ≥ 20 份）
- [ ] 使用 3 个渠道：直投（BOSS/猎聘）、内推（LinkedIn）、Web3 Job Board（CryptoJobs 等）
- [ ] 建立投递记录表（岗位/公司/投递时间/反馈）
- [ ] 技术面试准备：
  - [ ] 共识协议优缺点
  - [ ] 智能合约常见漏洞
  - [ ] DAG vs PBFT/HotStuff 实验结果解读
- [ ] 非技术准备：行业趋势、项目生态差异

## 第2周：面试集中期

- [x] 每天跟进投递反馈（发提醒邮件、加HR微信/LinkedIn）
- [x] 模拟面试：
  - [ ] 技术问答（安全性、复杂度分析）
  - [ ] 项目介绍（LightDAG、实验设计）
  - [ ] 行业理解（L2 发展、ZK 应用）

若出现"低回复率/低通过率"，优先定位问题来源：关键词匹配不足、项目叙事不清、答题案例薄弱。

- [ ] 复盘一轮（低回复率 → 优化简历关键词；低通过率 → 强化答题案例）
- [ ] 撰写一篇区块链研究/实验博客并发布（LinkedIn/Twitter/B站）

## 第3-4周：集中突破

核心规则：每周复盘必须量化三项指标（投递数量、回复率、通过率），并据此迭代下一周动作。

- [ ] 集中处理二面/三面（技术 + HR + Leader）
- [ ] 针对意向公司深挖研究资料（官网、GitHub、团队成员）
- [ ] 在行业社区活跃（DAO、TG群、黑客松）扩大曝光
- [ ] 每周复盘一次：
  - [ ] 投递数量 / 回复率
  - [ ] 面试数量 / 通过率
  - [ ] 调整策略
