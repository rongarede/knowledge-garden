# ERC-1155 多代币标准

## What（是什么）

- ERC-1155 是一个多代币标准，允许一个智能合约中同时管理多个 `tokenId`。
- 可同时支持：
  - 同质化代币（FTs，如游戏币）
  - 非同质化代币（NFTs，如装备）
  - 半同质化代币（SFTs，如门票）

## Why（为什么需要）

- 相比 ERC-20（单币）和 ERC-721（单 NFT），ERC-1155 更高效地管理多种资产。
- 改善点包括：
  - 批量转账更高效（节省 gas）
  - 合约更易扩展
  - URI 元数据统一

## How（如何工作）

- 每种资产由 `uint256 tokenId` 标识，映射其余额。
- 提供批量转账方法：
  - `safeTransferFrom()`：单 token 转移
  - `safeBatchTransferFrom()`：批量转移多个 tokenId
- 元数据 URI 使用占位模板：如 `https://example.com/{id}.json`
- 支持接收回调接口：
  - `onERC1155Received`
  - `onERC1155BatchReceived`

## Where（应用场景）

- 游戏资产系统（道具、武器、金币）
- NFT 卡包/盲盒平台
- DAO 凭证、活动门票系统
- Layer2/跨链桥批量资产桥接

## Who（生态参与者）

- Enjin：标准提出者
- OpenZeppelin：标准实现提供者
- OpenSea、Rarible：主流市场已支持
- GameFi 项目：多数链游已广泛采用

## 特点对比

| 特性 | ERC-20 | ERC-721 | ERC-1155 |
|---|---|---|---|
| 代币类型 | FT | NFT | 混合型（FT/NFT） |
| 唯一性 | 否 | 是 | 可选（支持两者） |
| 批量操作 | 否 | 否 | 支持 |
| Metadata URI | - | tokenURI | uri 模板 |
| gas 成本 | 高 | 高 | 低 |

## 示例函数接口

```solidity
function balanceOf(address account, uint256 id) external view returns (uint256);

function safeTransferFrom(
  address from,
  address to,
  uint256 id,
  uint256 amount,
  bytes calldata data
) external;

function safeBatchTransferFrom(
  address from,
  address to,
  uint256[] calldata ids,
  uint256[] calldata amounts,
  bytes calldata data
) external;
```
