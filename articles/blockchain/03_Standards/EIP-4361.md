# EIP-4361 格式设计原因

## 安全性

- 防钓鱼攻击：`domain` 明确签名适用的网站，用户可对照浏览器域名防止被伪造
- 防重放攻击：`nonce` + `issuedAt` + `expirationTime` 确保消息一次性且有时效
- 防跨链复用：`chainId` 限制签名仅在特定链有效

## 兼容性

- 钱包统一解析：所有兼容 EIP-4361 的钱包可用相同方式展示消息
- 跨平台标准：Web、移动端、硬件钱包验证逻辑一致

## 用户可读性

- 人类可理解的签名内容：`statement` 明确登录目的，例如"登录 Example DAO"
- 关键字段显式化：URI、链 ID、资源列表直接展示

## 开发便利性

- 后端解析简单：固定格式可用正则或解析库直接拆解字段
- 易于审计：安全检查时可验证字段完整性与正确性

## 与旧方式对比

| 方式 | 问题 | EIP-4361 改进点 |
|---|---|---|
| 任意文本签名 | 没有上下文，容易被重放 | 加入 `domain`、`chainId`、`nonce` 防止重放 |
| 交易签名 | 只能链上执行，无法链下认证 | 可链下验证身份 |
| 自定义 JSON 签名 | 不统一，钱包展示不可读 | 固定模板，跨钱包统一显示 |

## 总结

让用户看得懂、钱包能统一展示、后端易验证，并抵御签名滥用攻击。
