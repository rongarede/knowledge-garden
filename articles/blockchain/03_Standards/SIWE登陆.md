# SIWE 登录

## What（是什么）

- 基于以太坊账户的去中心化身份验证方式，由 **EIP-4361** 定义
- 用户通过用私钥签名一条标准化消息，证明自己对某个以太坊地址的控制权
- 无需用户名或密码即可在 Web 应用中完成身份认证

## Why（为什么用）

- 去中心化：无需依赖中心化账号数据库
- 安全：不存储密码，防止数据库泄露
- 防钓鱼/防重放：消息体包含 `domain`、`nonce`、`chainId` 等字段
- 跨应用通用：同一个以太坊地址可在多个 DApp 登录
- 无额外成本：签名为链下操作，不消耗 Gas

## When（何时使用）

- 登录 Web3 应用（NFT 市场、DeFi 平台、DAO 工具）
- 链下活动报名与权限验证
- Web3 社交平台的账户验证
- 跨应用单点登录（SSO）场景

## Where（在哪用）

- 用户端：支持以太坊的钱包（MetaMask、WalletConnect、Ledger 等）
- 服务端：DApp 后端、API 网关、Web3 社交/论坛平台
- 链下/链上结合：签名验证可链下完成，也可在智能合约中校验

## How（怎么实现）

1. 用户点击"登录/连接钱包"
2. 服务端生成 SIWE 标准消息（[[EIP-4361]] 格式，含域名、nonce、时间戳等）
3. 用户用钱包签名该消息（基于 EIP-191 / EIP-712）
4. 服务端用 `ecrecover` 或等效方法验证签名，确认签名者地址
5. 服务端为该地址建立会话（JWT、session ID），返回给用户
6. 用户后续请求携带会话 token 完成授权访问

```
┌─────────────┐
│  用户浏览器  │
└──────┬──────┘
       │ 点击"连接钱包/登录"
       ▼
┌─────────────┐
│  DApp 前端  │
└──────┬──────┘
       │ 请求后端生成 SIWE 消息
       ▼
┌─────────────┐
│  DApp 后端  │
└──────┬──────┘
       │ 生成 EIP-4361 标准化消息
       │ （含 domain、nonce、chainId、时间戳等）
       ▼
┌─────────────┐
│  用户钱包   │
└──────┬──────┘
       │ 用私钥签名消息（EIP-191/EIP-712）
       ▼
┌─────────────┐
│  DApp 后端  │
└──────┬──────┘
       │ 验签（ecrecover 地址 == 消息地址）
       │ 验证 nonce、时间戳、防重放
       ▼
┌─────────────────────────┐
│  登录成功               │
│  返回 JWT/Session 给前端 │
└─────────────────────────┘
```
