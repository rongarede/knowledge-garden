# ERC-721 标准介绍（4W1H）

## What（它是什么）

- ERC-721 是以太坊上第一个正式定义**非同质化代币（NFT, Non-Fungible Token）**的标准接口
- 每一个 ERC-721 代币在合约中都有唯一的 `tokenId`，表示其在链上的唯一性和不可互换性

### 定义了核心方法和事件，包括

- `ownerOf(tokenId)`：返回某个 token 的拥有者
- `safeTransferFrom(from, to, tokenId)`：安全转移 NFT 所有权
- `approve(to, tokenId)`：授权地址操作该 token
- `tokenURI(tokenId)`：返回 token 的元数据 URI
- `Transfer` / `Approval` 事件

- 依赖于 [[ERC165]] 接口识别标准（实现 `supportsInterface(bytes4)`）

### ERC-721 的标准组成

- `IERC721`：基础接口，定义所有权、转移、授权方法
- `IERC721Metadata`：扩展接口，提供 `name()`、`symbol()`、`tokenURI()`
- `IERC721Enumerable`（可选）：支持按索引查询所有 token
- 推荐继承层级：`ERC165 → ERC721 → ERC721URIStorage → ERC721Enumerable（可选）`

## Why（为什么需要）

- ERC20 无法表示"不可替代"的资产，如数字藏品、门票、身份等
- ERC-721 提供了一个通用标准，使钱包、市场、合约之间可以互操作 NFT
- 解决了链上表示"唯一物品"的通用问题，带来了：
  - 数字资产唯一性建模
  - 游戏资产、域名、艺术品的链上确权
  - 通证化的"拥有权"表达

### ERC-721 的关键特性与挑战

- 每个 token 都是唯一的、可识别的、可拥有的
- 标准化接口 → 钱包与市场统一识别
- 支持授权操作（`approve`, `setApprovalForAll`）

局限性：

- 不适合批量 mint（一次 mint 多个 token 成本高）
- `tokenURI` 通常托管在中心化 HTTP 服务（易失效）
- 无原生链上元数据压缩支持

## When（什么时候提出与使用）

- 提案编号：EIP-721
- 提出时间：2018 年 1 月
- 创始人：William Entriken 等人
- 成为 NFT 领域的基础标准，在 2018 后开始大规模应用

### 关键项目采用时间线

- 2018：CryptoKitties 使用预标准版本
- 2019-2020：OpenSea 成为通用 ERC-721 NFT 市场
- 2021+：NFT 热潮推动广泛应用，如 BAYC、Pudgy Penguins 等

## Where（在哪些系统中使用）

- NFT 市场平台：OpenSea、Blur、X2Y2
- 游戏资产：Decentraland（地块）、Axie Infinity（角色）
- 数字身份：ENS 域名、DID 身份 NFT
- 音乐 / 艺术：Catalog, Foundation, SuperRare
- 教育凭证：链上证书、任务徽章、SBT 等
- DAO：用作门票、会员卡、不可转让凭证等

### ERC-721 vs ERC-1155 vs ERC-20

| 特性 | ERC-721 | ERC-1155 | ERC-20 |
|---|---|---|---|
| 是否可替代 | 否 | 混合 | 是 |
| 多类型资产支持 | 单一 | 多 ID 支持 | 单种 token |
| 批量 mint / burn | 不优 | 高效 | 支持 |
| gas 成本 | 中等 | 更低（批处理） | 最低 |
| 典型应用 | 艺术品、证书 | 游戏道具、票据 | 代币、积分 |

## How（如何实现）

- 使用 Solidity 实现 `IERC721` 接口或继承 OpenZeppelin 的 `ERC721` 合约

### 推荐使用 OpenZeppelin 模板，自动处理

- `_safeMint()`, `_burn()`, `_setTokenURI()`, `approve()`
- 支持 `ERC721Enumerable` 和 `ERC721URIStorage` 扩展

### 示例合约片段

```solidity
contract MyNFT is ERC721 {
    constructor() ERC721("MyNFT", "MNFT") {
        _mint(msg.sender, 1);
    }
}
```

### 推荐模块搭配

- `Ownable`：控制权限
- `ERC721URIStorage`：支持自定义 tokenURI
- `ERC721Enumerable`：支持枚举所有 token
- `ERC165`：用于接口识别（supportsInterface）

### 安全实践建议

- 使用 `ERC721.safeMint()` 防止 NFT 被误发给无法接收的合约地址
- 始终启用 `supportsInterface()` 以便生态识别
- 建议加入 `Ownable` 管理 mint 权限，或设置白名单
- 可加入 `Pausable`（OpenZeppelin）以应对合约暂停场景

### ERC-721 进阶设计思路

- 动态 Metadata（如 SVG on-chain）
- Soulbound Token（SBT，非转移 NFT）
- 与 ERC-2981 版税标准结合（支持创作者收益）
- Token-bound Account（ERC-6551）：NFT 拥有独立钱包逻辑
- 与链下数据桥接（如 IPFS、Arweave、The Graph）
