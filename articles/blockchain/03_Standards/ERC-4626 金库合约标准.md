# ERC-4626 金库合约标准

## 背景

ERC-4626 出现之前，所有的金库合约比如 Yearn、Aave、Compound 各自设定各自的金库规范，并不统一。ERC-4626 出现之后，统一了金库合约的规范。

## ERC-4626 概念

ERC-4626 是为所有的金库合约定义的统一标准。ERC-4626 继承自 ERC-20，定义了用户存入资产、获得份额 token、用户换回资产+收益的一整个流程。

## ERC-4626 的核心方法

- `deposit`：用户指定存入多少资产，金库自动计算给多少份额
- `mint`：用户指定想要多少份额，金库自动计算需要存多少资产
- `withdraw`：用户指定取出多少资产，金库自动计算销毁多少份额
- `redeem`：用户指定赎回多少份额，金库自动计算换回多少资产

## ERC-4626 的公式

核心思路：先算份额单价，再根据存入或赎回的数量计算。

**份额单价**

```
份额单价 = 总资产 / 总份额
```

**存入**

```
获得份额 = 存入资产 / 份额单价
```

**赎回**

```
取回资产 = 赎回份额 × 份额单价
```

**例子**

1. Alice 存入 1000 USDC，获得 1000 份额
2. 金库赚了 500 USDC，现在总资产 1500，总份额 1000
3. 份额单价 = 1500 / 1000 = 1.5 USDC/份额
4. Bob 存入 900 USDC，获得 900 / 1.5 = 600 份额
5. Bob 赎回 600 份额，拿回 600 × 1.5 = 900 USDC
