# EIP-712 vs EIP-4361 对比

## 关系

- EIP-712：底层结构化数据签名标准，定义"怎么签"
- EIP-4361：应用层 SIWE 登录消息标准，定义"签什么"
- EIP-4361 可以选择用 EIP-712 或 EIP-191 来签消息

## 对比表

| 特性 | EIP-712 | EIP-4361 |
|---|---|---|
| 类型 | 底层签名格式标准 | 应用层协议标准 |
| 主要用途 | 对任意结构化数据安全签名 | 标准化 SIWE 登录消息 |
| 数据结构 | 自定义 types/domain/message | 固定的登录字段模板 |
| 安全特性 | 防重放（domainSeparator）、可读字段展示 | 防钓鱼、防重放、链/域名绑定 |
| 签名前缀 | `\x19\x01`（EIP-191 v0x01） | 可用 EIP-191 或 EIP-712 |
| 钱包展示 | 结构化字段表格 | 纯文本（EIP-191）或结构化表格（EIP-712） |

## 核心区别

- EIP-712 关注签名数据的结构化与哈希过程
- EIP-4361 关注登录消息的内容规范与安全字段

一句话总结：EIP-712 是签名"引擎"，EIP-4361 是登录"模板"，SIWE 用 EIP-712（或 EIP-191）来签它。
