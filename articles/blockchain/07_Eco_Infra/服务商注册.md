# 以太坊节点服务商（Ethereum Node Providers）

## What — 定义

节点服务商是提供远程访问 Ethereum 节点的第三方平台，支持开发者通过 RPC（Remote Procedure Call）接口与链交互。功能包括读取链上数据、部署智能合约、监听事件、交易广播等。典型接口：HTTP RPC / WebSocket / JSON-RPC。

## Why — 必要性

自建全节点资源消耗大（数百 GB 磁盘 + 网络同步）。节点服务商提供现成、稳定、可扩展的 API，方便快速开发与部署 dApp。便于在主网和测试网上使用统一的开发工具（如 Hardhat、Foundry、MetaMask 等）。

## When — 适用场景

- 本地开发或测试时使用公共测试网（如 Sepolia、Holesky）
- 生产环境部署合约或前端应用对链读取时
- 构建钱包、NFT 市场、DEX、MEV bot、链上数据可视化等服务
- 无法稳定运行 Geth / Erigon 全节点时的替代方案

## Where — 常见平台

- [Infura](https://infura.io)：主流平台，支持 IPFS，Consensys 支持
- [Alchemy](https://www.alchemy.com)：开发体验好，文档丰富，支持 Webhooks
- [QuickNode](https://www.quicknode.com)：响应速度快，支持多链和归档查询
- [Ankr](https://ankr.com)：支持多链 RPC 和 staking
- [Chainstack](https://chainstack.com)：企业级部署，支持自定义节点实例
- 其他：Blast API、RunNode、Pokt 等

## How — 使用方法

1. 注册平台账号，获取 API Key
2. 选择目标网络（如 Ethereum Mainnet / Sepolia）
3. 配置开发环境中的 RPC URL：

```env
RPC_URL=https://eth-sepolia.g.alchemy.com/v2/YOUR_API_KEY
```

4. 在 `ethers.js` 中使用：

```js
import { ethers } from 'ethers';
const provider = new ethers.JsonRpcProvider(process.env.RPC_URL);
```

5. Foundry 配置示例（foundry.toml）：

```toml
[profile.default]
rpc_endpoints = { sepolia = "https://eth-sepolia.g.alchemy.com/v2/YOUR_API_KEY" }
```
