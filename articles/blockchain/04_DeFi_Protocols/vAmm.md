# vAMM（Virtual AMM）

vAMM 实现杠杆的核心逻辑**并不是通过借真实资产来放大资金规模**，而是通过**使用保证金（Margin）+ 虚拟交易量（vToken）模拟更大的头寸**，从而实现"杠杆效果"。

## 杠杆原理（简化版）

**杠杆率（Leverage） = 虚拟交易头寸 / 实际投入保证金**

## 实现杠杆的机制解析（按模块拆解）

### 1. 保证金机制（Margin System）

- 用户提供真实资产作为保证金（如 100 USDC）
- 这部分资产不用于流动性池兑换，而是托管在合约中
- 合约根据保证金和用户指定的杠杆倍数来**计算用户可以开多大的虚拟头寸**

### 2. 虚拟交易池（vAMM）撮合

- vAMM 使用虚拟资产（如 vETH 和 vUSDC）构建虚拟池，维护恒定乘积：`vETH · vUSDC = k`
- 用户不会真的买卖 ETH，而是**模拟买入 vETH、卖出 vETH**

### 3. 杠杆开仓过程（多仓举例）

假设你有 100 USDC，想开 5 倍多仓（leverage = 5）：

**步骤一：计算可买的虚拟资产**

- 可操作的资金 = margin × leverage = 100 × 5 = 500 USDC
- 这 500 USDC 会用于模拟从 vAMM 中买入等值的 vETH
- 合约调用恒定乘积公式模拟你"花 500 USDC 买 vETH"，记录下你获得的 vETH 头寸

**步骤二：记录头寸信息**

```
Position {
  margin: 100,
  borrowed: 400, // 虚拟借贷
  position: +x vETH
}
```

你的头寸方向是**多仓（Long）**，你的盈亏由 vETH 价格变化决定。

### 4. 平仓与清算（保证系统风险控制）

**盈亏计算：**

- 平仓时再次模拟 vAMM 交易，将 vETH "卖回"换取 USDC
- 根据价格变化（即 vETH 所模拟价格的涨跌），你的 USDC 盈亏被计算出来

**清算逻辑：**

- 若你的账户净值跌至低于某一比例（如 10% 保证金），则会被系统强制平仓
- 模拟交易中亏损导致保证金不够支付欠债部分，系统通过清算机制回收剩余资产

## vAMM 实现杠杆 vs. 传统杠杆（对比）

| 项目 | vAMM 杠杆 | 传统杠杆（如 Aave 借贷） |
|------|----------|----------------------|
| **保证金** | 提供少量真实资产 | 提供抵押资产 |
| **借贷来源** | 无真实借贷，模拟交易 | 从资金池中借出真实资产 |
| **撮合方式** | 使用虚拟 AMM 价格 | 实际借贷与交易所价格挂钩 |
| **交易资产** | 虚拟资产（vETH、vUSDC） | 实际资产（ETH、DAI 等） |
| **清算方式** | 价格穿破触发模拟强平 | 资产抵押率低于阈值强平 |

## 示例：5 倍杠杆开多

起始：

```
margin = 100 USDC
leverage = 5
virtual buying power = 500 USDC
```

- 使用 500 USDC 在 vAMM 中买入 vETH，获得 ~0.25 vETH（取决于曲线）
- 假设 vETH 价格上涨 20%，新的虚拟价格上涨，你卖出 vETH，可得到 600 USDC
- 盈利 = 600 - 500 = 100，**实际收益 = 100%，但你只投入了 100 USDC → 杠杆效果实现**

## 小结

vAMM 杠杆的核心思想是：

- **用真实保证金托管风险**
- **用虚拟资产合约模拟交易**
- **价格通过虚拟池模拟 AMM 滑点逻辑**
- **盈亏最终以真实资产体现**
