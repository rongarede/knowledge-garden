# Compound 协议对比

## v2（多资产 + cToken 模型）

### 架构

- 每个资产一个独立市场（cToken 合约）
- Comptroller 统一风控与市场管理
- Price Oracle 提供价格数据
- 利率模型独立计算每个市场的借款/供应利率

### 核心机制

- 存款资产 → 获得对应 cToken
- 借款资产多样化，可借任何已上市资产
- 抵押物和存款资产可重合，抵押物可产生利息
- 利息通过 cToken 的 exchangeRate 累计

### 清算机制

- 计算健康度（可借额度 / 借款价值）
- 健康度 < 1 → 清算人偿还债务 → 获得折价抵押物

### 优点

- 多资产借贷灵活
- 抵押物可同时产生收益

### 缺点

- 风险暴露广（多资产价格波动）
- 治理和风控参数多，复杂度高
- 清算路径复杂

## v3（Comet：单一借款资产 + 无 cToken）

### 架构

- 单一借款资产市场（如 USDC）
- 抵押物可多样化（ETH、WBTC、LINK…）
- 所有逻辑集中在 Comet 合约（利率、清算、会计、奖励）
- Configurator 管理参数，内置奖励模块

### 核心机制

- 存入抵押物 → 借出该市场的唯一借款资产
- 抵押物不计息（仅作为担保品）
- 借款利息直接在 Comet 内部更新余额

### 清算机制

- 健康度 < 1 → 清算人偿还借款资产 → 获得抵押物
- 单一借款资产清算路径简单

### 优点

- 风险集中化（降低价格波动影响）
- 利用率和资本效率更高
- 清算更简单，治理负担小

### 缺点

- 抵押物不再产生利息
- 借款资产类型单一

## Compound v2 vs v3 区别表

| 对比项 | v2（多资产 + cToken） | v3（Comet：单一借款资产） |
|--------|----------------------|-------------------------|
| **市场结构** | 多市场，多资产，每个资产一个 cToken | 单一借款资产 + 多抵押物，无 cToken |
| **核心合约** | cToken（每市场） + Comptroller + Oracle | Comet（集成会计、利率、清算） + Configurator + Oracle |
| **借款资产** | 可借任意已上市资产 | 只能借该市场唯一的借款资产 |
| **抵押物收益** | 抵押物可产生利息（参与借贷） | 抵押物不计息（仅作担保） |
| **利息累计** | 通过 cToken 的 exchangeRate 累计 | Comet 内部直接更新余额 |
| **风险管理** | 多资产风险分散但复杂 | 风险集中化，可控性更高 |
| **清算逻辑** | 多资产清算路径复杂，需匹配借款资产与抵押物 | 单资产清算路径简单 |
| **治理复杂度** | 每市场独立参数配置（利率模型、抵押率、储备因子等） | 集中管理，参数更少 |
| **资本效率** | 资金分散在多个市场，利用率可能较低 | 资金集中在单一借款池，利用率高 |
| **激励机制** | 需额外机制分发 COMP | 内置原生奖励模块 |

## 核心差异速查

- 市场结构：v2 多市场，v3 单一借款资产 + 多抵押物
- 利息累计方式：v2 cToken exchangeRate，v3 直接在 Comet 内部更新余额
- 抵押物收益：v2 可产生利息，v3 不计息
- 风险管理：v2 多资产分散但复杂，v3 集中可控性高
- 清算复杂度：v2 多资产路径复杂，v3 单资产简单
- 治理负担：v2 每市场独立配置，v3 集中管理参数更少
