# Meme 平台 · 4+1 架构文档

## 项目概述

- 一个支持批量部署 Meme ERC20 Token 的平台，每个 Meme 是一个最小代理（Minimal Proxy）合约，逻辑由共享的 MemeToken 模板承载。
- 使用 UUPS 模式实现可升级性，采用工厂合约统一管理部署与 mint 流程，平台方可收取手续费。
- 技术基础：Solidity + OpenZeppelin（ERC20Upgradeable, Clones, UUPSUpgradeable）

## 1. 逻辑视图（Logical View）

### 模块划分

- MemeToken：ERC20 模板合约，支持初始化与受限 mint。
- MemeFactory：合约工厂，使用 UUPS 可升级代理部署，通过 clone 创建 MemeToken 实例。
- ERC1967Proxy：部署时用于 UUPS 模式的代理容器。

### 核心功能

- `deployMeme(symbol, totalSupply, perMint, price)`：创建一个 clone 合约并初始化 MemeToken 参数。
- `mintMeme(tokenAddr) payable`：用户支付费用铸造 perMint 数量的 MEME 代币；费用分账：1% 平台方，99% 给发行者。

## 2. 开发视图（Development View）

### 目录结构建议（Foundry 示例）

```
src/
  MemeToken.sol — 逻辑合约（UUPS 可升级）
  MemeFactory.sol — 工厂合约
script/
  Deploy.s.sol — 部署逻辑合约 + 工厂合约 + 初始化
test/ — 测试 deploy / mint / access 控制
```

### 工具链

- Foundry（forge, cast, anvil）
- OpenZeppelin Contracts Upgradeable v5

### 合约特性

- 使用 Initializable 控制构造流程
- 使用 onlyProxy, onlyOwner, UUPSUpgradeable 实现升级安全
- 支持未来扩展（如 Snapshot、白名单、mint 限制等）

## 3. 进程视图（Process View）

### 用户流程

- 项目方部署 MemeToken 实现合约 + MemeFactory 工厂合约（通过 UUPS 升级）
- 发行者调用 deployMeme() 创建 Meme token
- 用户调用 mintMeme() 支付 ETH → 自动调用 proxy.mint()
- 平台合约分账后将代币发送给用户

### 升级流程

- 项目方通过 upgradeTo() 升级 MemeFactory 或 MemeToken 实现合约

### 权限控制

- deployMeme 只允许外部用户调用
- mint 权限仅由工厂合约调用

## 4. 部署视图（Deployment View）

### 合约部署结构

- MemeToken 实现合约（UUPS 逻辑）
- ERC1967Proxy → 指向 MemeFactory（可升级）
- 工厂部署出的 clone 合约：MemeTokenClone1、MemeTokenClone2、...

### 存储结构（根据 EIP-1967）

- implementation：逻辑合约地址
- admin：UUPS 管理员地址（平台 owner）

### 外部交互组件（可选）

- Web3 前端：mint 页面 + 查询历史发行的 Meme
- Subgraph：跟踪所有 clone 地址、mint 次数等

## 5. 用例视图（Scenarios View）

### 使用场景 1：普通用户购买 Meme 代币

- 调用 mintMeme(tokenAddr) 并支付 ETH
- 获得 perMint 数量的代币，工厂合约处理分账

### 使用场景 2：Meme 创建者发行新项目

- 填写 symbol / totalSupply / perMint / price
- 调用 deployMeme() 部署新 MemeToken（Minimal Proxy）

### 使用场景 3：平台方升级 MemeToken 功能

- 开发新版本 MemeTokenV2（如加入 snapshot/mint 限额）
- 调用 upgradeTo(newImpl) 升级逻辑合约
