# 代币水龙头（Token Faucet 合约）【4W1H】

## What（它是什么）

- Token Faucet 是一种部署在区块链上的代币自动发放合约。
- 用户可通过调用 `requestTokens()` 接口定期免费领取少量测试代币（如 ERC20），用于开发、测试或教育用途。
- 一般与测试网代币（如 TST、USDT-mock）配合使用。
- 通常部署者需向 Faucet 合约提前转入足够的代币作为"储备池"。

## Why（为什么需要）

Faucet 合约是开发测试、教育教学、链上演示的重要工具，解决了新用户"没有初始代币"导致无法交互的问题。

它允许用户在没有资产的情况下获得小额代币用于：
- 测试 DApp 功能
- 演示交易流程
- 调试钱包或合约交互逻辑

同时也是黑客松、链上学习平台的常用基础设施，帮助降低用户参与门槛，提升开发与教学效率。

## When（什么时候使用）

在以下场景中常常部署 Faucet 合约：
- 开发阶段的本地链 / 测试网（Goerli, Sepolia, Anvil 等）
- 教学/演示平台（如区块链课程、链上打卡）
- Hackathon 活动发放测试代币
- 模拟治理/投票场景用 ERC20 投票权 Token
- 主网中也可作为领取激励（需加安全防护）

## Where（在哪些场景/系统中应用）

- 测试网代币发放：ERC20/ETH 水龙头
- GameFi：测试金币领取
- DAO：测试版投票权领取
- DeFi：模拟用户存款前的准备金领取
- 教育平台：用户打卡领取代币用于后续任务交互
- Frontend 演示：用户点击按钮领取少量 TST token，体验钱包交互

## How（如何实现）

合约核心组成：
- `IERC20 public token;`：绑定目标 ERC20 合约
- `mapping(address => uint256) lastRequestTime;`：记录每个用户的上次领取时间
- `uint256 faucetAmount; uint256 cooldownTime;`：发放额度与冷却时间

核心函数设计：
- `requestTokens()`：领取代币，需验证 cooldown 是否到期
- `canRequestTokens(address)`：返回是否能领取
- `getRemainingCooldown(address)`：返回剩余等待时间
- 管理函数（仅 owner）：
  - `setAmount()` 设置每次发放数量
  - `setCooldown()` 设置冷却时长
  - `withdraw()` 提取剩余代币

安全机制：
- 状态更新优先（符合 CEI 模式）
- 防止重入攻击
- 余额检查：防止代币不足发放失败
- 权限控制：使用 Ownable 限制敏感函数
