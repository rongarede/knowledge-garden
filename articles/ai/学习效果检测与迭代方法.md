# Task: 调研学习效果检测与迭代方法

## 背景

在学习 AI Fluency 课程的过程中，需要建立一套有效的学习检测机制，确保：
1. 学习内容真正内化，而非表面了解
2. 能够追踪学习进度和知识掌握程度
3. 建立可持续迭代的学习循环

## 调研目标

- [ ] 找到适合视频课程学习的效果检测方法
- [ ] 设计可量化的学习成果指标
- [ ] 建立知识迭代和复习机制
- [ ] 整合到 Obsidian 工作流中

## 调研方向

### 1. 学习效果检测方法

- [ ] **费曼技巧 (Feynman Technique)**
  - 用自己的话解释概念
  - 教别人是最好的学习
  - 检测点：能否用简单语言向他人解释 4D 框架？

- [ ] **主动回忆 (Active Recall)**
  - 闭卷回忆核心概念
  - 使用闪卡 (Flashcards) 测试
  - 工具：Anki, Obsidian Spaced Repetition 插件

- [ ] **间隔重复 (Spaced Repetition)**
  - 艾宾浩斯遗忘曲线
  - 在遗忘临界点复习
  - 设计复习时间表

- [ ] **概念映射 (Concept Mapping)**
  - 绘制知识关联图
  - 检验概念间的联系理解
  - 已有：Canvas 思维导图

- [ ] **实践应用检测**
  - 将 4D 框架应用到实际 AI 协作任务
  - 记录应用案例和效果
  - 对比学习前后的差异

### 2. 量化指标设计

- [ ] **知识掌握度评分**
  - 1-5 分自评每个概念
  - 定期重新评估

- [ ] **应用频率追踪**
  - 记录使用 4D 框架的次数
  - 追踪哪些技巧最常用

- [ ] **输出质量评估**
  - AI 协作产出的质量变化
  - 提示词迭代次数减少

### 3. 迭代机制设计

- [ ] **每日/每周复盘**
  - 学了什么？
  - 用了什么？
  - 下一步学什么？

- [ ] **知识卡片系统**
  - 将课程内容转化为原子化笔记
  - 每个概念一张卡片
  - 链接到 Zettelkasten

- [ ] **渐进式总结 (Progressive Summarization)**
  - 多轮精炼笔记
  - 高亮 → 加粗 → 摘要 → 重述

- [ ] **项目驱动学习**
  - 设定一个实际项目
  - 边学边用，用中学

### 4. 工具整合

- [ ] **Obsidian 插件调研**
  - Spaced Repetition (间隔重复)
  - Dataview (进度追踪)
  - Templater (学习模板)
  - Daily Notes (每日记录)

- [ ] **外部工具**
  - Anki (闪卡)
  - Notion (数据库追踪)
  - NotebookLM (AI 问答检测)

## 预期产出

1. **学习检测清单模板** - 每节课学完后的自检清单
2. **复习时间表** - 基于间隔重复的复习计划
3. **知识卡片库** - AI Fluency 核心概念卡片
4. **应用案例库** - 实践 4D 框架的案例记录
5. **迭代流程 SOP** - 可复用的学习迭代流程

## 初步想法

### 简单可行的检测方法

```
学完一节课后：
1. 关闭视频，写下 3 个核心要点 (主动回忆)
2. 用一句话向"小白"解释这节课内容 (费曼技巧)
3. 找一个实际场景应用所学 (实践检测)
4. 24h 后再次回忆，对比遗忘了什么 (间隔重复)
```

### 迭代循环

```
学习 → 回忆 → 应用 → 复盘 → 迭代
  ↑___________________________|
```
