# 函数调用的内存全景：栈、栈帧、递归与尾递归

> [!abstract] 核心观点
> 函数调用通过栈帧机制实现,如递归调用,栈帧数量超过了栈的内存空间,则会出现栈溢出

## 1. 进程内存布局

> 程序运行时，内存被分成哪几块？

分为堆、栈、DATA、BSS、TEXT, 五个区域, 堆跟栈是紧挨着,中间有一个空白区域,栈的空间如果超过了默认的大小上限,就是栈溢出, DATA 与 BSS, 功能有些类似, DATA用于存储已经赋值的变量, BSS用于存储没有赋值的变量, TEXT用于存储机器指令
低→高：Text → Data → BSS → Heap → ... → Stack）和**增长方向**（Heap ↑，Stack ↓）

## 2. 栈与栈帧

> 调用一个函数时，栈上发生了什么？

main 将要传给foo的值压入栈,再自动压入返回地址,再保存旧的RBP,再将新的RBP指向当前位置,再给foo分配局部变量

## 3. 函数调用链

> 多层调用时，栈帧如何叠加和回收？

main先调用 foo, 存入 foo 的返回地址,再存入foo 的局部变量, foo 再调用 bar 再存入 bar 的返回地址, 再 存入 bar 的局部变量, foo 的返回地址为 main+next , bar 的返回 地址为 foo+next, 

## 4. 递归

> 递归为什么会爆栈？

递归次数过多,每次递归相当于调用一个新的函数,每个函数都需要分配一个栈帧, 因此当递归层数过多时,栈空间会不够用

## 5. 尾递归与优化

> 为什么尾递归可以不增长栈？

尾递归会复用同一个栈的空间,类似于接力棒.尾递归再递归调用后不需要做任何事情,不需要保留返回地址,不需要保留状态,可以直接覆盖掉.所以编译器可以用 jump 代替call,直接服用当前帧

## 6. 与 EVM 的对比

> EVM 的"栈"和原生栈有什么不同？

EVM采用栈基架构,计算默认只能从栈顶存取,而x86采用寄存器架构,有一个寄存器用于存取数据进行计算. EVM的栈深只有1024,非常的浅,递归调用几乎不可用,quicksort 最坏递归深度 = n，n=170 就可能触及限制。
